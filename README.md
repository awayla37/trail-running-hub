# 🌲 Trail Hub - 越野跑专业装备与赛事仪表盘

> **项目状态**: 前端 MVP (Vibe Coding 范式)
> **核心定位**: 为越野跑“装备党”提供数字化管理、参数研究及垂直交流的一站式空间。

## 🎯 项目目标 (Goal)
针对目前越野跑装备资讯碎片化（散落在小红书、微信群如“超会买俱乐部”、抖音等）的痛点，Trail Hub 致力于：
1. **数字化管理**: 建立深度的个人“数字化装备间”，记录并追踪每一件硬核装备的物理参数与实战表现。
2. **知识沉淀**: 整合碎片化信息，形成可检索的装备百科与赛事复盘知识库。
3. **专业交流**: 满足跑者对于装备收集、技术参数（中底、大底、齿深等）研究以及基于实战的深度分享需求。

## 👥 用户画像 (Personas)
* **装备党 (Gear Enthusiasts)**: 核心需求是装备的数字化收集与参数横向对比。他们关注中底泡棉、大底配方、齿深等硬核数据。
* **赛事党 (Race Performance)**: 核心需求是根据赛事地形（如泥地、技术路面）从个人库存中选出最优配置，并记录赛事表现。

## 🛠️ 功能模块 (Modules)
* **Digital Closet (数字化装备间)**: 参数化管理（Brand, Model, Midsole, Outsole, Lug, Drop）及实战档案。
* **Race Calendar (赛事前哨)**: 中国内地主流赛程展示及强制装备 (Mandatory Kit) 自动核对。
* **Community Vision (交流社区)**: 整合散落在各平台的评价，建立基于真实实战的“评价即指南”社区。

## 📂 目录结构 (Structure)
* `/src/components`: 参数化 UI 组件（装备卡片、对比工具等）。
* `/src/data`: 硬核装备参数与赛事 Mock Data。
* `/src/store`: 基于 LocalStorage 的数据持久化方案。
