# MVP Plan（最小可行产品计划）

---

## 🎯 Goal（目标）

Validate the core assumption:

Do trail runners need gear recommendation based on routes?

验证核心假设：

越野跑者是否需要基于路线的装备推荐？

---

## 📦 Scope（范围）

### ✅ Include（包含）

* Route Database（路线数据库）

  * ~20 routes（赛事 + 训练）

* Gear Database（装备数据库）

  * ~30 trail running shoes

* Gear Input（装备输入）

  * Manual input
  * AI-assisted matching

* Recommendation Engine（推荐引擎）

---

### ❌ Exclude（不包含）

* User accounts（用户系统）
* Community features（社区）
* GPX upload（路线上传）
* AI terrain parsing（AI地形解析）

---

## 🧠 Core User Flow（核心用户流程）

Select Route
→ Input Gear
→ Get Recommendation

选择路线
→ 输入装备
→ 获取推荐

---

## 📊 Success Criteria（成功标准）

User can:

* Select a route
* Input gear easily
* Get a clear recommendation

用户可以：

* 选择路线
* 轻松输入装备
* 获得明确推荐

---

## ⚠️ Risks（风险）

* Users may not input gear
* Data quality issues
* Recommendation not useful

---

## 🧭 Next Step（下一步）

* Add GPX support
* Expand gear categories
* Introduce AI features
